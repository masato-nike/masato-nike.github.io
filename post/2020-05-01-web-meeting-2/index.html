Web会議システム構築（第2回 Linux（Ubuntu）へのJitsi Meetのインストール）

2020-05-01 Masato Nike

Linux(Ubuntu)にWeb会議システム『Jitsi Meet』をインストールします。
前提として、以下を実施しているものとします。

第1回 Linux（Ubuntu）サーバの構築

1. SSHの設定

(1) アプリケーションの一覧を表示してください。
(2) [端末]をクリックさせてください。

以下、端末にコマンドを入力することになります。

(3) SSHを利用できるようにします。以下のコマンドを入力してください。

sudo apt update
sudo apt upgrade
sudo apt install openssh-server
sudo /etc/init.d/ssh restart
sudo ufw status
sudo ufw allow proto tcp from 192.168.100.0/24 to any port 22
sudo ufw enable

(4) rootでのログインを拒否するように設定します。

【コマンド】

sudo vi /etc/ssh/sshd_config

【編集内容】

PermitRootLogin no

(5) 以下のコマンドを入力してください。

sudo systemctl restart sshd
sudo systemctl status sshd

以上で、SSHの設定は終了です。

以下は端末で操作してもよいですし、TeraTermProなどで操作しても構いません。

2. Jitsi Meetインストール事前準備

(1) 以下のコマンドを入力してください。

sudo ufw allow in 443/tcp
sudo ufw allow in 10000:20000/udp
sudo ufw status
sudo wget https://download.jitsi.org/jitsi-key.gpg.key
sudo apt-key add jitsi-key.gpg.key
sudo sh -c "echo 'deb https://download.jitsi.org stable/' >> /etc/apt/sources.list.d/jitsi-stable.list"
sudo apt update
sudo apt install openjdk-8-jre-headless
sudo apt install nginx

3. Jitsi Meetのインストール

(1) 以下のコマンドを入力してください。

sudo apt install jitsi-meet

(2) インスト―ルの途中で以下の画面になったら、«ホスト名.ドメイン名»を入力してください。

(2) インスト―ルの途中で以下の画面になったら、2つ目の方を選択してください。

以上で、『Jitsi Meet』のインストールは終了ですが、まだ起動はできません。
httpsでアクセスするため、SSLサーバ証明書の設定が必要になります。
